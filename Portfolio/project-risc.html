A high-efficiency 16-bit RISC processor featuring a streamlined instruction set, single-cycle execution, and integrated memory controller, designed and verified using Verilog HDL and Xilinx Vivado.

Project Overview

This project involved the design and verification of a high-efficiency 16-bit RISC (Reduced Instruction Set Computing) processor. Developed using Verilog HDL, the architecture features a streamlined 15-instruction set and an integrated memory controller designed to optimize data throughput and memory utilization.

The processor follows the classic RISC philosophy of simplicity and efficiency, focusing on a compact instruction set with streamlined control logic. This approach enables faster clock cycles, simpler control units, and predictable performance characteristics ideal for embedded systems and educational applications.

Project Highlights

  • Single-Cycle Execution: All instructions execute in one clock cycle for deterministic performance
  • 15-Instruction Set: Comprehensive yet minimal instruction set covering all essential operations
  • Integrated Memory Controller: On-chip controller regulates data flow between processor and memory
  • Modular Architecture: Clean separation of concerns with dedicated functional units
  • Fully Verified: Comprehensive test benches ensure correctness of all instructions and interactions

System Architecture & Design

The processor design follows a classic RISC philosophy, focusing on a compact instruction set and streamlined control logic. The architecture emphasizes simplicity and regularity, enabling faster design iterations and easier verification.

Core Components

Arithmetic Logic Unit (ALU)

Performs arithmetic and logical operations including addition, subtraction, AND, OR, XOR, and shifts

Control Unit

Decodes opcodes and generates control signals to orchestrate operations across all functional units

Program Counter (PC)

Maintains current instruction address and supports branching operations with conditional and unconditional jumps

Register File (GPRS)

8-register general-purpose register file for variable storage and operand passing between instructions

Memory Controller

Manages data transfers between processor core and memory modules with efficient arbitration

Data Path

Multiplexers and buses connecting all functional units for efficient data flow

Design Philosophy

The processor design embodies core RISC principles:

  • Simplicity: Minimal instruction set reduces complexity of control unit and decoder
  • Regularity: Uniform instruction format simplifies instruction decoding and execution
  • Speed: Single-cycle execution enables predictable timing and high clock frequencies
  • Orthogonality: Operations and addressing modes are cleanly separated and independent

Signal Flow

  1. Instruction Fetch: PC selects instruction from memory
  2. Instruction Decode: Control Unit interprets opcode and generates control signals
  3. Register Read: Operands fetched from register file
  4. ALU Execute: Arithmetic or logical operation performed
  5. Register Write: Result written back to destination register
  6. PC Update: Next instruction address calculated

Instruction Set Architecture (ISA)

The processor supports a streamlined 15-instruction set designed to cover all essential operations while maintaining decoder simplicity. All instructions follow a uniform format for consistent control logic.

Instruction Type Operation Execution Time
ADD Arithmetic Rd = Rs + Rt (Register addition) 1 cycle
SUB Arithmetic Rd = Rs - Rt (Register subtraction) 1 cycle
ADDI Immediate Rd = Rs + Immediate (Add immediate) 1 cycle
AND Logical Rd = Rs & Rt (Bitwise AND) 1 cycle
OR Logical Rd = Rs | Rt (Bitwise OR) 1 cycle
XOR Logical Rd = Rs ^ Rt (Bitwise XOR) 1 cycle
SLL Shift Rd = Rs << Shamt (Shift Left Logical) 1 cycle
SRL Shift Rd = Rs >> Shamt (Shift Right Logical) 1 cycle
LW Load Rd = Memory[Rs + Offset] (Load Word) 1 cycle
SW Store Memory[Rs + Offset] = Rt (Store Word) 1 cycle
BEQ Branch if (Rs == Rt) PC = PC + Offset (Branch if Equal) 1 cycle
BNE Branch if (Rs != Rt) PC = PC + Offset (Branch if Not Equal) 1 cycle
JMP Jump PC = Address (Unconditional Jump) 1 cycle
JAL Jump R7 = PC+1; PC = Address (Jump and Link) 1 cycle
NOP Control No Operation (Pipeline synchronization) 1 cycle

Single-Cycle Execution Model

All 15 instructions complete within a single clock cycle, eliminating pipeline stalls and making execution time completely predictable. This design choice prioritizes simplicity and determinism over instruction throughput, making it ideal for control-intensive and real-time applications.

Verification Strategy

Comprehensive verification ensures functional correctness through multiple testing levels:

  • Unit Testing: Each module (ALU, Control Unit, etc.) tested independently
  • Integration Testing: Module interactions validated through component-level tests
  • Functional Testing: All instructions executed with diverse operand combinations
  • Edge Case Testing: Branch conditions, overflow, boundary values tested
  • Performance Testing: Timing margins verified, critical paths identified

Test Coverage

The testbench architecture includes:

  • Instruction execution validators for all 15 opcodes
  • Memory access pattern checkers for LW/SW
  • Branch prediction and PC update validators
  • Register file write-back correctness checkers
  • Timing constraint verifiers

Design Challenges & Solutions

Single-Cycle Timing

Challenge: Achieving single-cycle execution within timing constraints.
Solution: Optimized critical path through careful data flow design and combinatorial logic placement.

Instruction Decoding

Challenge: Generating all control signals from instruction encoding within one cycle.
Solution: Regular instruction format enables simple combinatorial decoder without complex prioritization.

Memory Controller

Challenge: Integrating memory access into single-cycle pipeline.
Solution: Synchronous design with pre-decoded memory access enables seamless integration.

Data Dependencies

Challenge: Handling true data dependencies in single-cycle architecture.
Solution: No forwarding needed due to single-cycle execution; dependencies handled by programmer discipline.

Control Flow

Challenge: Efficient branch resolution in single cycle.
Solution: Pre-calculated branch targets and condition evaluation in instruction phase.

Modularity

Challenge: Maintaining clean modular design while ensuring timing closure.
Solution: Hierarchical Verilog with clear interfaces and optimized internal implementations.

Project Overview

This project introduces a cost-effective and energy-efficient metal detection system designed for multi-domain applications, including household appliances, recycling, and industrial quality control. Unlike traditional sensors like VLF (Very Low Frequency) or Pulse Induction systems, this innovative approach identifies metals by analyzing real-time changes in the frequency and amplitude of an LC-resonant circuit.

The system combines analog signal conditioning with advanced machine learning to achieve high precision metal classification while maintaining exceptional energy efficiency. This makes it ideal for battery-powered and permanently-installed applications where low power consumption is critical.

Project Highlights

  • Detection Latency: Less than 150ms from signal acquisition to classification
  • Power Efficiency: Significantly lower consumption than traditional metal detection systems
  • Real-Time Processing: Integrated Raspberry Pi for instantaneous feedback and logging
  • Feature Engineering: Frequency shift identified as the most discriminative feature

LC Resonant Circuit Principle

Resonance Frequency Formula

Resonant Frequency

$$f_r = \frac{1}{2\pi\sqrt{LC}}$$
Where L is the inductance (Henry) and C is the capacitance (Farad). The system operates based on the principle that metallic objects near the sensing coil alter its inductance, causing a measurable shift in the resonant frequency.

Detection Mechanism

When a metallic object approaches the sensing coil, its magnetic properties interact with the coil's electromagnetic field. This interaction changes the effective inductance (L) of the circuit, which directly affects the resonant frequency according to the formula above.

Different metals have different magnetic permeabilities, causing varying degrees of inductance change. By measuring these frequency shifts and amplitude variations, the system can classify the type of metal present.

System Stages

  • Sensing: Custom-wound copper coil generates a resonant oscillation at a baseline frequency
  • Signal Conditioning: Frequency and amplitude variations are measured and digitized by Arduino
  • Data Acquisition: Raw sensor data is collected and preprocessed for ML classification
  • Classification: ML model analyzes features to determine metal type

Signal Flow Process

  1. Oscillation Generation: LC circuit oscillates at its natural resonant frequency
  2. Metal Detection: Metallic object alters inductance, shifting frequency and amplitude
  3. Signal Conditioning: Analog circuits amplify and filter the signal for digital processing
  4. Data Acquisition: Arduino samples signal at 10kHz using ADC for high resolution
  5. Feature Extraction: Frequency shift, amplitude, and other metrics computed in real-time
  6. Classification: Raspberry Pi processes features through trained MLP model
  7. Output: Metal type identified with confidence score and logged to database

Core Innovation: Frequency & Amplitude-Based Classification

The fundamental breakthrough of this system is achieving high-accuracy metal classification using only frequency (ΔF) and amplitude (ΔA) variations from the LC resonant circuit, eliminating the need for an external distance sensor. This approach simplifies hardware, reduces costs, and improves reliability while maintaining exceptional classification accuracy.

By leveraging sophisticated machine learning algorithms trained on comprehensive electromagnetic signatures, the system can distinguish between multiple metal types and even identify precious metals with near-perfect precision all from two core measurements.

  • Cross-Validation: 5-fold validation ensures robustness and prevents overfitting
  • Hyperparameter Tuning: Grid search optimization for learning rate, batch size, and network architecture

Hardware Architecture & Signal Flow

The system employs a sophisticated two-stage processing architecture combining analog signal acquisition with digital machine learning inference. The dual-processor design optimizes cost, power consumption, and processing capability while maintaining real-time performance.

Signal Processing Pipeline

Stage 1: Analog Signal Acquisition (Arduino Uno)

  • Oscillator: NE555-based LC Resonant Circuit continuously oscillates at characteristic frequency (~34 kHz)
  • Coil Interaction: Proximity of metal alters LC resonance, producing frequency shift (ΔF) and amplitude change (ΔA)
  • Analog-to-Digital Conversion: Arduino ADC samples conditioned signals at 10 kHz for digital processing
  • Feature Extraction: Real-time calculation of frequency deviation and amplitude magnitude
  • Data Transmission: Arduino sends extracted features (ΔF, ΔA) to Raspberry Pi via UART serial interface

Stage 2: Digital Classification (Raspberry Pi 3 B+)

  • Feature Reception: Receives frequency and amplitude variations from Arduino Uno
  • Model Selection: Loads appropriate ML model (Model A for general classification or Model B for gold detection)
  • Inference Execution: TensorFlow/Keras or Scikit-learn performs real-time classification inference (<150ms)
  • Confidence Scoring: Returns predicted metal type with confidence percentage
  • Logging & Output: Results logged to CSV and displayed on CLI interface
  • Latency: Total end-to-end response time: ~150ms for real-time user feedback

Assembly & Integration

The hardware implementation follows standard embedded systems practices with careful attention to signal integrity and electromagnetic compatibility:

  • PCB Design: Multilayer board with ground planes to minimize noise and EMI
  • Component Placement: Signal path optimized with analog circuits away from digital logic
